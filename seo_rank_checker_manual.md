# SEO 排名检测工具 v1.2 — 产品功能操作手册

## 概述

SEO 排名检测工具是一款基于桌面图形界面（tkinter）的关键词排名自动检测软件。它从远程接口拉取关键词列表，通过 Selenium 驱动真实浏览器访问各大搜索引擎，检测目标域名在指定关键词下的搜索排名，并将结果实时上报至远程接口，同时支持本地导出。

**核心功能一览：**

- 支持 Google / Google TW / Yahoo / Bing / 百度 五大搜索引擎
- 自动从远程 XML 接口拉取关键词及目标域名
- 多引擎并发翻页检测，结果实时显示在表格中
- 每条检测完成后自动 GET 提交至结果接口
- 支持导出 CSV / JSON 结果文件
- 实时彩色日志面板，日志同步写入本地文件

---

## 界面结构

工具窗口自上而下分为五个区域：

```

├──────────────────────────────────────────┤
│  检测配置面板（引擎选择、参数、接口设置）    │
├──────────────────────────────────────────┤
│  工具栏（开始/停止 + 导出 + 进度条）        │

```

## 功能模块详解

### 一、检测配置面板

配置面板是所有检测参数的入口，位于顶栏正下方，包含四行设置。

#### 1. 搜索引擎选择

提供五个复选框，默认勾选 Google：

| 选项 | 引擎地址 | 默认 |
|------|---------|------|
| Google | google.com | ✅ 勾选 |
| Google TW | google.com.tw | 未勾选 |
| Yahoo | tw.search.yahoo.com | 未勾选 |
| Bing | bing.com | 未勾选 |
| 百度 | baidu.com | 未勾选 |

可同时勾选多个引擎，工具会对每个关键词逐一在所有勾选引擎中检测。

#### 2. 参数设置

| 参数 | 说明 | 默认值 | 范围 |
|------|------|--------|------|
| 每引擎翻页数 | 每次检测最多翻几页搜索结果 | 1 | 1 ~ 20 |
| 关键词间隔（秒） | 每个关键词检测完后的等待时间，实际值会在 ±0.5s 范围内随机浮动 | 10.0 | 1 ~ 120 |
| 数量限制 | 从接口拉取的关键词最大条数，0 表示不限制 | 0 | 0 ~ 9999 |
| 日志级别 | 控制 GUI 日志面板显示的最低级别 | INFO | DEBUG / INFO / WARNING / ERROR |

#### 3. 关键词接口

- **接口地址**：默认为 `https://www.san-seo.com.tw/seed.php`，可手动修改为其他 XML 接口地址。
- **加载关键词按钮**：点击后在后台线程请求接口，解析 XML 中的关键词 ID（`<id>`）、关键词（`<k>`）和目标域名（`<u>`），去重后加载入内存。加载完成后按钮旁显示"✓ 已加载 N 条"。

XML 接口格式示例：
```xml
<root>
  <sp>
    <id>123</id>
    <k>SEO优化</k>
    <u>example.com</u>
  </sp>
</root>
```

#### 4. 提交结果接口

- **接口地址**：默认为 `https://www.san-seo.com.tw/get.php`，可修改。
- 每条检测完成后，工具自动以 GET 方式提交以下参数：

| 参数 | 说明 |
|------|------|
| `kid` | 关键词 ID |
| `type` | 引擎代码（GE / G / Y / B / BD） |
| `on` | 排名位次（未找到时为 0） |
| `creatdate` | 检测日期，格式 `YYYY-MM-DD` |

### 二、工具栏

#### 开始检测

点击红色"▶ 开始检测"按钮启动检测任务。启动前会做两项校验：

1. 必须已加载关键词（否则弹出提示"请先点击【加载关键词】"）
2. 必须至少勾选一个搜索引擎（否则弹出提示）

启动后，按钮变灰不可点击，"停止"按钮激活。

#### 停止检测

点击"⏹ 停止"按钮发送停止信号，当前正在进行的单次搜索完成后任务中断，不会强行终止浏览器进程。停止后日志会输出已完成数量和提交统计。

#### 导出 CSV

将当前所有检测结果导出为 CSV 文件。弹出系统保存对话框，默认文件名格式为 `seo_rank_YYYYMMDD_HHMMSS.csv`。文件使用 UTF-8 BOM 编码，可直接用 Excel 打开。

CSV 字段说明：

| 字段 | 含义 |
|------|------|
| keyword | 检测关键词 |
| domain | 目标域名 |
| engine | 搜索引擎名称 |
| rank | 排名位次（未找到时为"未找到"） |
| page | 排名所在页码 |
| url | 命中的结果 URL |
| note | 备注（如"触发验证码"、"前N页未找到"） |
| submitted | 提交结果接口是否成功（✓ / ✗） |
| checked_at | 检测时间戳 |

#### 导出 JSON

与 CSV 导出相同逻辑，保存为 UTF-8 JSON 文件，默认文件名为 `seo_rank_YYYYMMDD_HHMMSS.json`，数据结构为对象数组，字段同上。

#### 日志目录

点击后用系统文件管理器打开日志文件所在目录（`logs/`），方便批量查阅历史日志。

#### 清空结果

清除结果表格中的所有行、清空内存结果列表、重置进度条。不影响已导出的文件和已写入的日志。

#### 进度条

工具栏右侧显示实时进度，格式为"已完成 / 总任务数"，绿色进度条展示百分比。总任务数 = 关键词数 × 勾选引擎数。

---

### 三、结果表格

结果表格实时展示每条检测任务的状态和结果，支持上下左右滚动。

#### 表格列说明

| 列名 | 含义 |
|------|------|
| # | 关键词 ID |
| 关键词 | 检测的关键词文本 |
| 目标域名 | 需要查找排名的域名 |
| 引擎 | 搜索引擎名称 |
| 排名 | 检测到的排名位次（未找到显示"未找到"） |
| 页码 | 排名结果所在的页码 |
| 命中 URL | 匹配到的搜索结果链接（截取前 80 字符显示） |
| 备注 | 附加说明信息 |
| 已提交 | 是否成功提交至结果接口（✓ / ✗） |
| 检测时间 | 本条结果完成时的时间戳 |

#### 行颜色含义

| 颜色 | 含义 |
|------|------|
| 蓝色字 / 蓝底 | 检测中 |
| 绿色字 / 绿底 | 找到排名 |
| 橙色字 / 橙底 | 未找到排名 |
| 红色字 / 红底 | 检测出错 |

#### 双击行操作

双击结果表格中的任意一行，会将该行"命中 URL"字段的内容复制到系统剪贴板，状态栏同步显示"已复制 URL: ..."。

---

### 四、运行日志面板

位于结果表格下方（可拖动分割线调整高低比例），深色背景展示实时日志输出。

#### 日志颜色说明

| 颜色 | 日志级别 |
|------|---------|
| 灰色 | DEBUG |
| 浅灰 | INFO |
| 黄色 | WARNING |
| 红色 | ERROR |
| 紫色 | CRITICAL |

日志面板最多保留 2000 行，超出后自动删除最早的内容。

#### 日志面板操作按钮

- **打开日志文件**：用系统默认文本编辑器打开当前会话的日志文件（`logs/seo_YYYYMMDD_HHMMSS.log`）。
- **清空日志**：仅清空 GUI 面板中的显示内容，不影响磁盘日志文件。

---

### 五、状态栏

窗口底部一栏，左侧显示当前状态文字，右侧显示最近一次状态更新的时间。

| 状态 | 颜色 | 触发场景 |
|------|------|---------|
| 就绪 | 灰色 | 启动初始状态 |
| 正在加载关键词... | 蓝色 | 点击加载关键词后 |
| 关键词加载完成，共 N 条 | 绿色 | 关键词加载成功 |
| 检测中... | 蓝色 | 检测任务运行中 |
| [引擎] 关键词 → 排名: N | 蓝色 | 每条结果完成时 |
| 正在停止... | 红色 | 点击停止后 |
| ✅ 检测完成！共 N 条 | 绿色 | 全部任务完成 |
| 已清空结果 | 灰色 | 点击清空结果后 |

## 典型操作流程

以下是一次完整检测任务的标准步骤：

**第一步：配置检测参数**

打开软件后，在"检测配置"面板中勾选需要使用的搜索引擎（至少一个），根据需要调整翻页数、关键词间隔和数量限制。

**第二步：加载关键词**

确认"关键词接口"地址正确后，点击红色"⬇ 加载关键词"按钮。等待接口返回数据，加载完成后面板右侧显示"✓ 已加载 N 条"，结果表格中自动预填所有待检测行（状态为"待检测"）。

**第三步：确认提交接口**

检查"提交结果接口"地址是否正确，若需要上报到自定义接口可在此修改。

**第四步：开始检测**

点击工具栏的"▶ 开始检测"按钮。工具按关键词顺序逐一检测，每个关键词会依次在所有勾选引擎中搜索，结果实时更新在表格中，日志面板同步输出详细信息。

**第五步：查看结果并导出**

检测完成后，状态栏显示"✅ 检测完成！"。此时可点击"📥 导出 CSV"或"📥 导出 JSON"将结果保存到本地文件，也可在表格中双击任意行复制命中 URL。

**第六步（可选）：查看日志**

点击"📂 日志目录"或日志面板中的"打开日志文件"按钮，可查看详细的 DEBUG 级运行记录，便于排查异常。

---

## 日志文件

每次启动软件会在 `logs/` 目录下自动创建一个新的日志文件，文件名格式为：

```
seo_YYYYMMDD_HHMMSS.log
```

日志记录所有 DEBUG 及以上级别的信息，包括引擎初始化、关键词检索过程、每页解析结果、提交状态、任务耗时等，是排查问题的主要依据。

## 注意事项

**关于 Bing 引擎**：Bing 使用 `undetected_chromedriver` 以有头模式（非无头）运行，首次使用时会打开一个可见的浏览器窗口进行预热，属于正常现象，无需干预。

**关于验证码**：当 Google 等引擎检测到异常流量时，单条结果的备注列会显示"触发验证码"，排名记为未找到。建议适当增大"关键词间隔"参数以降低触发频率。

**关于停止任务**：点击停止后工具会在当前搜索动作结束后才真正停止，不会立即中断。任务结束后（无论正常完成还是停止）浏览器驱动会自动关闭释放。

**关于结果提交**：每条检测结果（包括未找到的条目）都会自动提交，未找到时 `on` 参数传 0。若提交失败，"已提交"列显示"✗"，可通过日志文件查看具体错误原因。

**关于数量限制**：数量限制参数在请求关键词接口时生效，设为 0 表示拉取接口返回的全部条目。修改此参数后需重新点击"加载关键词"才能生效。
